## 9. Definition of Done

### 9.1 Story-Level DoD

A story is considered "Done" when:

- [ ] Code is written and committed to feature branch
- [ ] Code is reviewed by at least one team member
- [ ] All review feedback addressed
- [ ] Unit tests written (coverage > 80%)
- [ ] Integration tests written (if applicable)
- [ ] All tests passing locally
- [ ] All tests passing in CI
- [ ] No regressions in existing tests
- [ ] Code follows style guide (ESLint passing)
- [ ] TypeScript compilation successful
- [ ] API documentation updated (if API changed)
- [ ] User documentation updated (if user-facing)
- [ ] Verified on staging environment
- [ ] Product Owner sign-off (if applicable)

### 9.2 Sprint-Level DoD

A sprint is considered "Done" when:

- [ ] All P0 stories completed
- [ ] All P1 stories completed or deferred with agreement
- [ ] No critical bugs open
- [ ] Successfully deployed to staging
- [ ] Sprint review completed
- [ ] Sprint retrospective completed
- [ ] Next sprint backlog ready
- [ ] Documentation updated
- [ ] Demo completed for stakeholders
